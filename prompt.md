правила валидации, которые должны быть соблюдены во ВСЕМ проекте(бд, фронт, бэк)

USER:
id - уникальный, создаётся автоматически на уровне бд
username - not null, min 4, max 16
password - not null, min 4, хэшируется в бд

NOTE:
id - уникальный, создаётся автоматически на уровне бд
title - not null, min 1, max 32
content - not null, min 1, max 255
category - not null, NoteCategory enum(EnumType.STRING в базе)
creationDate - not null, создаётся автоматически при создании заметки, и при редактировании заметки
userId - привязан обязательно к какому-то существующему юзеру

если какой-то эндпоинт может нарушить логику, то должна вернуться соответсвующая ошибка,
а не просто 500 ошибка сервера. Сделать четкую валидацию на уровне API и фронта. 
Если на фронте пользователь пытается отправить невалидные данные - вывести предупреждение
об ошибке
